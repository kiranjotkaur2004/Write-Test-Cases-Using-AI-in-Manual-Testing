# Write Test Cases Using AI in Manual Testing

## Overview

This project demonstrates how Artificial Intelligence can assist Manual Testers in generating comprehensive test cases from user stories, requirements, and feature descriptions. The AI analyzes functional requirements and automatically creates structured test cases, helping QA teams improve productivity, consistency, and test coverage.

## Features

* Generate manual test cases from user stories
* Create positive, negative, and boundary test scenarios
* Generate preconditions, test steps, and expected results
* Produce Jira-compatible test case formats
* Improve requirement traceability
* Reduce manual effort in test design

## Problem Statement

Manual test case creation can be time-consuming and prone to missed scenarios. AI helps QA engineers quickly generate comprehensive test cases while maintaining testing standards and coverage.

## AI-Powered Workflow

1. Enter a user story or requirement.
2. AI analyzes the feature and business rules.
3. AI generates:

   * Functional Test Cases
   * Negative Test Cases
   * Boundary Test Cases
   * Validation Scenarios
   * Jira-Compatible Test Cases
4. QA Engineer reviews and refines the generated output.

## Sample User Story

"As a logged-in user on an e-commerce website, I can select the product with the lowest price and add it to my cart if it is available in inventory."

| Test Case ID | Description                                       | Preconditions                        | Steps                                              | Expected Result                                               |
